Technology

For the entire project I tried to stay as basic as possible. When possible I didn't fork and customized projects, used the default models in Django and so on - this is not the usual way I develop, and it's sometimes a little limiting, but I want to give a project that can perhaps also teach a little bit about Django, using open source and general technologies.

The site is build using Python/Django, my currently favorite language and web framework. It was built using a lot of good open source technologies like django-redis-cache (for caching of anonymous users and sessions), and obviously dependent on RedisPy client for that as well. For social oauth stuff Pairup is using django-social-auth, but I didn't like the way it interacted with Facebook, so specifically for that I used python-sdk client and interacted with the Facebook API directly. Since many people didn't want to use social networks to log in, I added django-registration that allows for email authentication registration process. The only thing I did fork and changed a little was django-avatar, since I wanted to optionally display the images imported from above social networks, if they existed.

To admin and log I use a slightly forked version of mine of django-sentry - the extra things I added there are really minor, you can use the default version. Just to sum up a couple of Django plugins used when developing the site: django-extensions and obviously django toolbar

But this is not where it ends: For the basics of the html I used Paul Irish's HTML5 Biolerplate, and the equivalent email version of HTML5 Email biolerplate - both are the results of years or experience dealing with all the hardships of browsers and email readers. I hardly code direct CSS anymore, but use Scss through Compass)
